配置中间件 → 部署项目到服务器 → 监控 → 性能压测

1. 中间件服务器（MySQL / Redis / Kafka）

填写各服务所在机器的 IP 和端口，项目部署后会连接这些地址。

2. 项目服务器（项目将部署到此机器）

3. 项目代码仓库

4. OpenClaw（AI 生成 MySQL 案例）

用于从页面添加新问题时，调用 OpenClaw Agent 自动生成可运行案例。需本地或内网可访问 OpenClaw Gateway。

部署

从 GitHub 克隆项目 → 编译 → 生成配置 → rsync 到项目服务器。

                  真实业务场景
                  
电商平台大促秒杀：双11 活动开始瞬间每秒 5000+ 请求，应用每次新建 MySQL 连接且未归还，max_connections=500 约 10 秒后新请求全部报 Too many connections，用户界面显示「系统繁忙，请稍后再试」，活动被迫暂停。

                

                

                  
1
业务场景：应用未正确复用连接，不断创建新连接而不释放。

                  
2
现象：新连接报错 Too many connections；Threads_connected 接近 max_connections。

                  
3
问题：MySQL 连接数达到上限。

                  
4
解决方案建议：调整 max_connections；使用连接池；修复连接泄漏。
